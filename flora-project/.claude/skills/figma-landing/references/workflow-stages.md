# Поетапний конвеєр збірки

Етапи виконуються **строго по черзі**. Кожен має «ворота» — поки вони не
закриті, наступний етап не починаємо.

## Етап 1 — Семантична розмітка (без CSS)
- Розмітити сторінку (або узгоджену частину) семантичним HTML5 за макетом.
- Жодного CSS: ні `<style>`, ні `<link rel="stylesheet">`.
- Зображення — заглушки-шляхи (`./images/...`), бо порожній `src` = помилка.
- **Ворота:** Nu-валідатор → 0 помилок (попередження розглянути).

## Етап 2 — Формат + ассети
- Прогнати Prettier; звірити з конвенціями codeguide.co.
- Підключити `modern-normalize`.
- Зібрати SVG-спрайт з іконок; підготувати растр x1/x2 (retina); фавікон.
- **Ворота:** код відформатовано; ассети на місці й оптимізовані.

## Етап 3 — CSS (Mobile First)
- Витягнути токени з Figma → CSS custom properties у `:root` (єдине джерело).
- Стилі писати від мобільної ширини, нарощувати через `min-width` на
  брейкпоінтах макета.
- Жодних «магічних чисел» — лише змінні/значення з макета.
- **Ворота:** збіг із макетом на кожному брейкпоінті (звірка накладанням
  рендера Figma) + валідний CSS (jigsaw.w3.org).

## Етап 4 — JS-поведінка
- Реалізувати рівно те, що вимагає макет/ТЗ: бургер-меню, слайдери (Swiper),
  модалки (закриття X/backdrop/Escape, блок скролу), запити на бекенд,
  пагінацію тощо.
- Доступність: керування з клавіатури, фокус, стани (disabled/active/hover).
- **Ворота:** поведінка відповідає макету; немає помилок у консолі.

## Етап 5 — Фінал
- README (назва, опис, технології, як запустити/розгорнути).
- Фінальна оптимізація графіки й завантаження.
- Повний прохід: HTML-валідатор, CSS-валідатор, чек-лист вимог по секціях.
- **Ворота:** усі пункти чек-листа пройдені; обидва валідатори чисті.
