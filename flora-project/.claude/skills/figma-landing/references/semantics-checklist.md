# Семантика і валідність — правила

Семантична розмітка = теги за їх призначенням відповідно до специфікації.
Валідатор перевіряє **синтаксис і відповідність специфікації**, а не семантику —
семантику звіряй головою за html.spec.whatwg.org.

## Документ
- `<!DOCTYPE html>`, `<html lang="…">`, `<meta charset="UTF-8">`,
  `<meta name="viewport" content="width=device-width, initial-scale=1.0">`,
  `<title>`.

## Теги за змістом
- Каркас: `<header>`, `<nav>`, `<main>` (один на сторінку), `<section>`,
  `<article>`, `<aside>`, `<footer>`.
- Заголовки: рівно один `<h1>`; ієрархія без пропусків (h2 → h3 …).
- Списки однотипних елементів — `<ul>`/`<ol>` з `<li>` (навігація, товари,
  соцмережі, переваги).
- Цитати/відгуки — `<figure>` + `<blockquote>` + `<figcaption>` (автор).
- Контактна інформація — `<address>` (телефон через `tel:`, пошта через
  `mailto:`).
- **`<a>` vs `<button>`:** посилання ведуть на адресу/якір (`<a href>`); дії
  (відкрити модалку, довантажити, перемкнути слайд) — `<button type="button">`.

## Типові помилки валідатора (уникати)
- `src=""` — помилка; `src` має бути непорожнім (став шлях-заглушку).
- `<img>` без `alt` — помилка; декоративним зображенням — `alt=""`.
- Дублікати `id` на сторінці.
- Кнопка без `type` усередині форми сабмітить її — для не-сабміту став
  `type="button"`.
- Невалідна вкладеність (напр. блок усередині `<p>`).
- На етапі розмітки — жодного CSS-підключення.

## Доступність (і добра семантика водночас)
- `aria-label` на навігації та іконкових кнопках (без видимого тексту).
- `aria-labelledby` — прив'язка `<section>` до її заголовка.
- `target="_blank"` завжди з `rel="noopener noreferrer"`.

## Інструменти
- HTML: `validator.w3.org/nu/` (онлайн) або `vnu` (Java, офлайн). Без npm.
- CSS: `jigsaw.w3.org/css-validator/`.
